# ADR-006: Isabelle Independence

**Status:** ACCEPTED

## Decision

Isabelle verification must not import or assume truth assertions generated by Lean.

Isabelle must independently reconstruct or reprove every designated load-bearing theorem.

## Reconstruction Requirement

Isabelle may share **certificate data** with Lean (raw polynomial coefficients, computed identities) only if **each system verifies independently**.

Isabelle must NOT:

- Import a Lean `.olean` file
- Assume a Lean axiom definition
- Accept "Lean said this is true" as verification
- Use Lean's theorem names as external oracles

## Independent Reconstruction Process

For each designated theorem:

1. State the exact theorem in Isabelle syntax
2. Independently prove it using Isabelle's libraries
3. Compare the final statement to Lean's statement
4. Record any semantic mismatch
5. If mismatch found, investigate which system has a weakened assumption

## Load-Bearing Theorems Requiring Isabelle Reconstruction

Phase 2+, to be determined:

- Formal derivative laws (chain rule, product rule, power rule)
- Jacobian of identity and composition
- Determinant multiplicativity
- Dimension-one explicit inverse (if proved in Lean)
- Any reduction used to bound the search space in Phase 4

## Mismatch Protocol

If Isabelle proof succeeds where Lean version fails:

→ Investigate Lean proof for hidden sorry or incomplete case split

If Lean proof succeeds where Isabelle version fails:

→ Investigate Lean for hidden axiom or weakened assumption

If proofs have same length but different conclusions:

→ Stop and escalate to independent mathematical review

## Consequence

Isabelle is not a backup checker. Isabelle is an **independent auditor**.

Disagreement between Lean and Isabelle indicates a real problem, not a minor discrepancy.
